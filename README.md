# 🤖 AI Voice Assistant (DeepSeek 极速架构版)

## 📌 项目简介

专为黑客马拉松（Hackathon）设计的**纯前端无服务器 (Serverless)** 智能语音对话终端。
项目彻底抛弃了传统的后端中间件路由，利用现代浏览器的底层多媒体 API 直连国内的 DeepSeek 认知大模型。通过高度工程化的前端有限状态机（FSM）管理，实现了毫秒级的精准环境拾音与全双工智能对话打断体验。

## 🚀 核心架构与黑科技特性

*   **端侧静默侦测 (VAD)**：深度融合 `Web Audio API` 与原生 `AnalyserNode`，通过持续的事件轮询（`requestAnimationFrame`）精准监控麦克风的物理频段。当系统侦测到环境静音超过了硬编码的物理阈值（`800ms`）时，会自动触发断句动作并对状态机上锁，全程无需依赖任何显式的唤醒词。
*   **双模式动态热切换引擎**：
    *   🔒 **安全互斥模式 (Mutex Lock)**：通过强制挂起 `video.pause()` 并切断 `SpeechRecognition` 数据流，实现了严密的半双工物理级硬件隔离。此模式能够彻底斩断在路演现场因为扬声器外放声音被再次录入而引发的死循环回音（Echo Loop）。
    *   ⚡ **全双工打断模式 (Full-Duplex)**：引入 W3C 推荐的 `AbortController` 底层网络异常熔断技术。当侦测到用户在 AI 播报期间强行抢话时，系统能在物理网络协议层瞬间截断未完成的 HTTP 数据接收流，并同步调用系统级的 `speechSynthesis.cancel()` 强行清空操作系统的音频渲染缓冲池。
*   **极致轻量级的降维部署**：整个应用脱离了 Python / Node.js 后端的重度依赖包袱，仅凭一个前端 Vue 单页面（SPA）即可在沙盒中跑通所有的 API 鉴权与长链接通信业务。

## 🛠️ 极速启动指南

本工程基于轻量且极速的 `Vite` 引擎构建，对系统底层的依赖要求极低。

```bash
# 1. 下载工程依赖
npm install

# 2. 启动本地开发容器
npm run dev
```

在终端输出成功的地址后（通常为 `http://localhost:5173/`），在弹出的本地浏览器（**强烈推荐 Chrome 内核**）中，授权页面的麦克风与摄像头白名单权限。最后在暗黑科幻风格的初始化密码弹窗中，键入您个人的 DeepSeek API Key（以 `sk-` 开头）即可建立神经元链接。

## 🛡️ 安全合规与凭证隔离

本项目完美符合开源极客社区的代码安全审计规范。系统在工程架构设计上**严禁**将明文形式的调用凭据（API Key）硬编码进任何环境配置文件（如 `.env`）或 Vue 源码内部。
用户的鉴权凭据将在前端输入后，通过浏览器的 `localStorage` 原生接口被强加密，并持久化隔离在当前域名的本地存储安全沙盒中。即使页面刷新授权依然有效，且绝不会向第三方发生网络侧漏。
